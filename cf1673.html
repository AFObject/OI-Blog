1673B A Perfectly Balanced String?

Medium- *114514结论

算法

五花八门。

你找一个长度最短且首尾字符相同的，此时首尾字符出现的次数为 \(2\)，那么中间的次数只能是 \(1\) 咯，必须每个都轮一遍。轮不到的寄。

实现

这个 last 的用法还挺妙的。

int T;
char s[200003];
bool e[27];
int last[27];

int main() {
    scanf("%d", &T);
    while (T--) {
        scanf("%s", s);
        const int len = strlen(s);
        for (int i = 0; i < 27; i++)
            e[i] = false;
        for (int i = 0; i < len; i++)
            e[s[i] - 'a'] = true;
        int kind = 0;
        for (int i = 0; i < 26; i++)
            kind += e[i], last[i] = -1;
        for (int i = 0; i < len; i++) {
            if (last[s[i] - 'a'] != -1) {
                if (i - last[s[i] - 'a'] != kind) {
                    puts("NO");
                    goto end;
                }
            }
            last[s[i] - 'a'] = i;
        }
        puts("YES");
        end: continue;
    }
    return 0;
}

1673C Palindrome Basis

Medium- *114514背包DP

算法

考虑设 \(f(i,j)\) 为在前 \(i\) 个回文数中（第 \(i\) 个为 \(p_i\)），凑出来数 \(n\) 的方案数。则有：

\[f(i,j)=f(i-1,j)+f(i,j-p_i) \]

考虑省去一维。

\[f(j)\gets f(j)+f(j-p_i) \]

\(f(0)=1\)。

关于循环顺序：两维的时候你爱咋地咋地，但一维的时候省去了 \(i\) 这维，所以大前提就是在虚拟遍历 \(i\) 只不过没在转移方程里体现出来，\(i\) 必须放在外层，否则就会对 \(f(j)\) 的值产生影响。

实现

signed main() {
    T = read(); f[0] = 1;
    for (int k = 0; k < P; ++k)
        for (int i = p[k]; i <= 50000; ++i)
            f[i] = (f[i] + f[i - p[k]]) % MOD;
    while (T--)
        printf("%lld\n", f[read()]);
}
